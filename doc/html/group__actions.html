Detailed Description

These classes are used to represent the various actions that can be executed on the simulator.

Actions are divided into two groups. Some actions are designed to be used in custom card databases, making third-party developers able to create their own cards without requiring from them to dive into the simulator code. However, special actions like ending the turn (EndTurnAction), making a minion attack (AttackAction), playing a card (PlayAction) or summoning a minion (SummonMinionAction) are not to be used by third-party developers, as they depict the core mechanics of the game engine itself.
